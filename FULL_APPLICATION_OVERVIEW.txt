STAY AND TREK - APPLICATION OVERVIEW

- Stay and Trek is a web and mobile platform for exploring hiking trails in Ireland.
- The project combines spatial search, trail information, nearby accommodation, weather data, and routing in one system.
- It was built to support both trail discovery and trip planning, rather than only showing trail locations on a map.

MAIN PURPOSE

- help users find trails by location, distance, county, and map interaction
- support broader trip planning by showing nearby accommodation and route options
- provide a mobile-friendly way to browse trails while keeping the web platform for deeper map analysis and admin tasks

MAIN FEATURES

- interactive web map for trail discovery
- trail list and detail views on web and mobile
- proximity search and bounding box search
- polygon-based town search in the advanced mapping section
- dashboard and town analytics features
- nearby accommodation linked to trails
- route generation between trail start points and accommodation
- live weather integration for trails and towns
- moderator review flow for user-submitted trail descriptions
- accessibility options such as contrast and text scaling

WHY THESE FEATURES WERE CHOSEN

- The trail start point was used as the main reference point because it reflects how many walkers begin a trip in practice.
- In many cases a user may drive to the trailhead, complete a walk, and then travel onward to accommodation, so routing between the trail and nearby stays made practical sense.
- The accommodation feature was included because walking trips are often planned as part of a wider visit, especially for people travelling across counties rather than doing a short local walk.
- The polygon search and analytics features were included to support wider spatial questions, not just individual trail lookup.

SYSTEM OVERVIEW

- The backend is built with Django and Django REST Framework.
- Spatial data is stored in PostgreSQL with PostGIS.
- The web application uses Django templates, JavaScript, and Leaflet.
- The mobile app is built with React Native and Expo.
- The project is deployed to Google Cloud Run, with Cloud SQL used for the cloud database.

WORKFLOW

- local development is run through Docker
- cloud deployment uses Google Cloud Run and Cloud SQL
- local Docker and cloud environments use separate databases
- mobile submissions enter a moderation workflow before approval

TESTING AND VALIDATION

- backend API and integration tests were added to support reliability
- the deployment pipeline includes a test gate before cloud deployment
- manual testing was also used for map interaction, mobile flow, admin moderation, and routing behaviour

CURRENT LIMITATIONS

- local and cloud environments must be kept clearly separate
- some admin tasks still depend on manual moderation steps
- the mobile app is still a lighter companion to the full web platform
- some features could be improved further for usability and consistency across devices

FUTURE DEVELOPMENT

- expand filtering by county, distance, route type, and user preference
- extend user features such as favourites and saved trails
- improve automated testing and CI/CD coverage further
- continue mobile development and wider user testing
- improve route planning between trails and accommodation
- add caching for external API data
- extend accessibility options further
