                    STAY AND TREK - FULL APPLICATION OVERVIEW

COMPREHENSIVE APPLICATION ARCHITECTURE

Project Overview
================================================================================
Stay and Trek is a full-stack Irish hiking discovery platform with a Django REST 
API backend, PostgreSQL PostGIS database, interactive web mapping interface, and 
React Native mobile app for cross-platform access.


APPLICATION STRUCTURE
================================================================================

1. BACKEND (Django + GeoDjango)
-----------------------------------

Framework Stack:
- Django 5.2 + Django REST Framework 3.16
- GeoDjango for spatial/geographic queries
- PostGIS extension for PostgreSQL
- DRF Spectacular for API documentation
- Python 3.11

Core Apps:

trails_api - Main API
  - REST Endpoints serving trail, town, POI, accommodation, and boundary data
  - Custom Managers for spatial queries (radius search, bounding box, nearest points)
  - Serializers including GeoJSON support via djangorestframework-gis
  - Spatial Filtering using DjangoFilterBackend

advanced_js_mapping - Advanced Features
  - City polygon analysis
  - Session tracking for analytics
  - Enhanced city models with demographics

authentication - User Management
  - User registration/login
  - JWT token authentication

dashboard - Analytics & Statistics
  - Trail statistics
  - User favorites management

API ENDPOINTS
================================================================================

Trail Endpoints:
  GET    /api/trails/                    # List all trails (paginated)
  POST   /api/trails/                    # Create new trail
  GET    /api/trails/<id>/               # Trail detail view
  GET    /api/trails/geojson/            # GeoJSON format
  GET    /api/trails/paths/geojson/      # Trail path geometry
  GET    /api/trails/search/             # Full-text search
  GET    /api/trails/within-radius/      # Spatial radius query
  GET    /api/trails/bbox/               # Bounding box query
  GET    /api/trails/stats/              # Trail statistics
  GET    /api/trails/weather/<id>/       # Weather data

Town Endpoints:
  GET    /api/trails/towns/geojson/      # Towns as GeoJSON
  GET    /api/trails/nearest-town/       # Find nearest town
  GET    /api/trails/load-towns/         # Load all towns
  GET    /api/trails/weather-town/       # Town weather

POI Endpoints:
  GET    /api/trails/pois/                          # List POIs
  GET    /api/trails/pois/type/<poi_type>/          # Filter by type
  GET    /api/trails/pois/near-trail/               # POIs near trail
  GET    /api/trails/pois/radius-search/            # Radius search

Boundary Endpoints:
  GET    /api/trails/boundaries/                            # List boundaries
  GET    /api/trails/boundaries/<id>/trails-crossing/       # Trails crossing boundary
  GET    /api/trails/boundaries/<id>/trails-near/           # Trails near boundary
  GET    /api/trails/boundaries/county/<name>/trails/       # Trails by county
  GET    /api/trails/spatial-analysis/summary/              # Spatial analysis

Accommodations Endpoints:
  GET    /api/trails/accommodations/                 # List accommodations
  GET    /api/trails/accommodations/geojson/         # GeoJSON format
  GET    /api/trails/accommodations/near-trail/      # Near specific trail
  GET    /api/trails/accommodations/near-town/       # Near towns
  GET    /api/trails/accommodations/by-county/       # Filter by county
  GET    /api/trails/accommodations/stats/           # Accommodation statistics

Authentication & Documentation:
  POST   /api/token/                     # JWT token obtain
  POST   /api/token/refresh/             # Token refresh
  GET    /api/schema/                    # OpenAPI schema
  GET    /api/docs/                      # Swagger UI
  GET    /api/redoc/                     # ReDoc documentation

DATA MODELS & RELATIONSHIPS
================================================================================

Trail Model:
  Fields:
    - trail_name, description, activity
    - county, region, nearest_town
    - distance_km, difficulty, elevation_gain_m
    - start_point (PointField), path (MultiLineStringField)
    - dogs_allowed, parking_available, facilities, trail_type
    - status (verified/pending/rejected/missing)
    - favorited_by (ManyToMany with User)
    - source_url, created_at, updated_at

  Methods:
    - within_radius(center_point, radius_km)
    - in_bounding_box(bbox)
    - nearest_to_point(point, limit=10)
    - duration_hours (computed from difficulty)

Town Model:
  Fields:
    - name, country, town_type (capital/city/town/village)
    - location (PointField)
    - population, area, gdp_per_capita
    - unemployment_rate, elevation_m

  Computed Properties:
    - latitude, longitude
    - population_density

PointOfInterest Model:
  Fields:
    - name, poi_type, description
    - location (PointField with geography)
    - county, region, phone, website, opening_hours
    
  Types:
    - parking, cafe, restaurant, attraction, viewpoint
    - toilet, shelter, picnic, information, accommodation

GeographicBoundary Model:
  Fields:
    - name, boundary_type, geom (PolygonField)
    - description, established_date

  Types:
    - county, region, protected_area, national_park
    - nature_reserve, forest, river, stream

  Methods:
    - trails_crossing()
    - trails_within()
    - trail_intersection_points()

Accommodation Model:
  Fields:
    - name, location (PointField)
    - accommodation_type (hotel/hostel/guesthouse/etc)
    - county, region, price_category
    - facilities, phone, website, opening_hours

DATA FLOW
================================================================================

Trail Discovery Flow:
  User → Web/Mobile App → Axios/Fetch → Django REST API
    ↓
  GeoDjango Spatial Query → PostgreSQL + PostGIS
    ↓
  Serializer → GeoJSON Response
    ↓
  Leaflet/React Native Maps → Display on map/list

Proximity Search Flow:
  User clicks map → JavaScript captures coordinates
    ↓
  AJAX POST to /api/trails/within-radius/
    ↓
  TrailManager.within_radius() → Distance calculation
    ↓
  GeoJSON features returned
    ↓
  Leaflet renders markers with popups

Mobile App Flow:
  Expo → React Native Components
    ↓
  Axios HTTP Client → API calls
    ↓
  Response parsing → State management
    ↓
  React Native Maps → UI Rendering

WORKFLOW SUMMARY
================================================================================

1. USER INTERACTION (Web)
   - User visits web interface at localhost:8000
   - Loads interactive map with Leaflet.js
   - Clicks on map to set proximity search radius
   - JavaScript AJAX sends POST to /api/trails/within-radius/

2. USER INTERACTION (Mobile)
   - User launches Expo app (via Expo Go or build)
   - Browses trails in list format
   - Taps trail for detailed view
   - App uses Axios to fetch from /api/trails/ endpoints

3. BACKEND PROCESSING
   - Django REST Framework receives request
   - URL router directs to appropriate view
   - View uses TrailManager spatial queries
   - GeoDjango translates to PostGIS SQL

4. DATABASE QUERY
   - PostGIS calculates distances using spatial indexes
   - Returns matching trails with geometries
   - Database returns results in milliseconds

5. SERIALIZATION
   - Serializers convert Django models to JSON/GeoJSON
   - GeoFeatureModelSerializer handles geographic data
   - Response includes coordinates and properties

6. RESPONSE TO CLIENT
   - Web: JavaScript receives GeoJSON, renders on Leaflet
   - Mobile: React Native parses JSON, displays in list/map

7. VISUALIZATION
   - Web: Leaflet shows markers, popups, boundaries
   - Mobile: React Native Maps shows clustered pins

TROUBLESHOOTING GUIDE
================================================================================

Common Issues:

1. PostGIS not found
   - Ensure docker-compose.yml running postgis/postgis image
   - Check GDAL_LIBRARY_PATH in settings.py
   - On macOS: brew install gdal geos proj

2. Static files not loading
   - Run: python manage.py collectstatic
   - Check STATIC_URL and STATIC_ROOT in settings.py
   - Verify nginx volume mounts in docker-compose.yml

3. API returning empty results
   - Check database migrations ran: python manage.py migrate
   - Verify data population: python manage.py create_sample_trails
   - Check query parameters and filters

4. Spatial queries slow
   - Verify spatial indexes created
   - Check query with EXPLAIN ANALYZE
   - Monitor database connections
   - Consider query optimization

5. Mobile app connection issues
   - Verify API endpoint URL in mobile config
   - Check CORS settings in Django
   - Ensure firewall allows requests
   - Test with curl: curl -H "Content-Type: application/json" http://localhost:8000/api/trails/
